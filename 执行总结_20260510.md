## API 端点

### 认证

- `POST /api/auth/login` - 用户登录
- `GET /api/auth/captcha` - 获取验证码
- `POST /api/auth/logout` - 用户登出

### 预警

- `GET /api/warning/sse` - SSE 实时连接
- `GET /api/warning/list` - 预警列表
- `PUT /api/warning/:id/read` - 标记已读
- `PUT /api/warning/read-all` - 全部已读
- `GET /api/warning/unread-count` - 未读数量
- `POST /api/warning/create` - 创建预警
- `POST /api/warning/check` - 运行预警检查

### 报表

- `GET /api/reports/hazard` - 隐患统计报表
- `GET /api/reports/work-permits` - 作业票统计报表
- `GET /api/reports/training` - 培训统计报表
- `GET /api/reports/risk-tips` - 风险智能提示
- `GET /api/reports/dashboard-full` - 综合 Dashboard

### 作业票（7类）

- `GET /api/safework/:type/list` - 列表查询
- `GET /api/safework/:type/:id` - 详情查询
- `POST /api/safework/:type` - 创建作业票
- `PUT /api/safework/:type` - 更新作业票
- `PUT /api/safework/:type/updateStatus` - 更新状态
- `DELETE /api/safework/:type/:id` - 删除作业票

类型前缀：
- `highwork` - 高处作业
- `restrictedwork` - 受限空间作业
- `electricwork` - 临时用电作业
- `blindinfo` - 盲板抽堵作业
- `brokenInfo` - 断路作业
- `hoistingwork` - 吊装作业
- `earth` - 动土作业

## 常见问题

### 1. 登录报 404 错误

**原因**：Nginx 反向代理未生效

**解决**：
1. 检查 Nginx 是否启动
2. 查看 `D:\phpstudy_pro\Extensions\Nginx1.15.11\conf\vhosts\0localhost_80.conf` 配置
3. 确保包含：
   ```nginx
   location /api/ {
       proxy_pass http://127.0.0.1:3001;
   }
   ```

### 2. 数据库连不上

**原因**：MySQL 版本或密码错误

**解决**：
1. 确认 MySQL 已切换到 8.0.12
2. 检查 `backend\.env` 文件中的数据库配置
3. 默认密码：`root`

### 3. H5 端无法访问

**原因**：端口被占用或 Node.js 未安装

**解决**：
1. 检查 Node.js 是否安装：`node -v`
2. 检查端口 3002 是否被占用：`netstat -ano | findstr :3002`
3. 修改 `frontend-h5\vite.config.js` 中的端口配置
