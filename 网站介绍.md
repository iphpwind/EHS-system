## 一、系统概述

安全生产管理系统是一套面向工业企业的**综合安全管理平台**，基于工业互联网架构设计，涵盖隐患排查、作业票管理、应急管理等核心安全业务场景，支持 PC 端和移动端 (H5) 多端访问。

**核心特点**：
- ✅ **全栈自研**：前端 Vue3 + 后端 Node.js + MySQL 8.0
- ✅ **多端适配**：PC 管理端 + H5 移动端 + 大屏展示
- ✅ **模块化设计**：可按需启用/禁用功能模块
- ✅ **权限精细**：RBAC 权限模型，支持数据权限控制
- ✅ **实时预警**：SSE 推送，ECharts 可视化

### 4.2 核心功能说明

#### 4.2.1 首页驾驶舱
- **数据可视化**：使用 ECharts 5 展示安全趋势、隐患分布、KPI 指标
- **实时更新**：支持 SSE (Server-Sent Events) 推送
- **权限适配**：根据角色显示不同的数据范围

#### 4.2.2 作业票管理 (7类特殊作业)
- **完整流程**：创建 → 安全措施 → JSA 分析 → 签字 → 审批 → 作业 → 验收
- **PDF 生成**：自动生成标准化作业票 PDF 文档
- **二维码**：每个作业票生成唯一二维码，便于现场扫码核查

#### 4.2.3 隐患管理
- **隐患上报**：支持图片上传、定位、分级分类
- **整改跟踪**：整改通知 → 整改实施 → 验收闭环
- **统计分析**：隐患类型分布、部门排名、趋势分析

#### 4.2.4 培训教育
- **线上线下结合**：支持线下培训记录和线上课程学习
- **考试管理**：题库 → 试卷 → 考试 → 成绩统计
- **证书管理**：培训合格自动生成电子证书

#### 4.2.5 RBAC 权限模型
- **用户-角色-权限**：三级权限控制
- **数据权限**：支持按部门/区域过滤数据
- **模块开关**：管理员可以启用/禁用功能模块

## 七、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 页面白屏 + 401 | localStorage 有过期 token | F12 → Application → 清除 localStorage → 刷新 |
| 后端自动停止 | 未捕获的异常导致进程崩溃 | 已添加全局异常处理 + PM2 自动重启 |
| 前端构建内存溢出 | Vite 构建吃内存 | `node --max-old-space-size=4096 node_modules/vite/bin/vite.js` |
| 验证码 404 | 路径错误 | 正确路径是 `/api/code`，不是 `/auth/captcha` |
| H5 登录失败 | token 字段名错误 | 确认读取 `res.data.access_token` |
| PC 构建后 API 404 | 用了 `/dev-api` 前缀 | 必须用 `npm run build:prod` |

## 八、开发规范

### 8.1 前端构建规范

**PC 前端**：
```bash
# ⚠️ 必须用 build:prod，否则 API 前缀会是 /dev-api
npm run build:prod
```

**H5 前端**：
```bash
# ⚠️ 构建前必须删除旧 dist 和 WWW\h5
rmdir /s /q dist
rmdir /s /q D:\phpstudy_pro\WWW\h5
npm run build
```

### 8.2 后端开发规范

- 所有 API 必须经过 `authenticateToken` 中间件认证
- 使用 `asyncHandler` 包装异步路由，避免未捕获异常
- 数据库操作使用 `getConnection()` 获取连接，用完必须 `release()`
- 错误使用 `AppError` 类抛出，由 `errorHandler` 中间件统一处理

### 8.3 代码提交规范

